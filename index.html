Manila Shared File Systems Integration

Select shared file systems capacity, configure multi-tenant networking via DHSS (Driver Handles Share Servers), and review ports and security boundaries.

Driver Handles Share Servers (DHSS) Mode

DHSS Enabled (DHSS = True)
Implications: Manila automatically provisions dedicated Share Servers (virtual machines, containers, or NetApp SVMs) dynamically inside tenant-isolated Neutron networks.
Pros: High security, complete L3 isolation between tenants (DESC/NCA Compliant).
Cons: High storage array overhead, complex setup, requires NetApp cluster API integration.
DHSS Disabled (DHSS = False)
Implications: Manila routes all traffic to pre-existing, static storage network interfaces. Separation is managed at the filesystem layer.
Pros: Lower latency, lightweight setup, compatible with Native CephFS and simple NFS.
Cons: Tenants share the same storage IP; network isolation must be handled externally.

Select Manila Storage Backends (Select one or more)

CephFS Native
Direct Ceph filesystem mount inside VM. Extremely fast, requires tenant access to Ceph OSD network directly.
CephFS via NFS-Ganesha
CephFS mounted via NFS-Ganesha gateways. Safer, hides Ceph network behind an NFS proxy layer.
NetApp ONTAP NAS
Provisions NFS/CIFS shares on dynamic SVMs (DHSS=True) or a static SVM (DHSS=False) over standard network interfaces.
